Integration Guide

Allow your users to sign in using their DIDChain Identity. Instead of passwords, users authenticate using their digital wallet, providing you with a verified Web3 Identity and a Reputation Score directly from the blockchain.

1Add the Button

Include our professional CSS and the authorization link in your login page.

<!-- Include Styles -->
<link rel="stylesheet" href="https://did-blockchain.onrender.com/css/did-button.css">

<!-- Sign In Button -->
<a href="https://did-blockchain.onrender.com/authorize?redirect_uri=YOUR_URL" class="did-connect-button">
  <span class="did-icon">⬡</span>
  Sign in with DIDChain
</a>

2Handle the Callback

After the user approves the signature, we redirect them back to your redirect_uri with a token.

https://your-site.com/callback?did_token=JWT_TOKEN_HERE&address=0x...

Extract the did_token from the URL query parameters to verify it in the next step.

3Verify the Token (Backend)

To prevent spoofing, your backend must verify the token using our endpoint.

const response = await fetch('https://did-blockchain.onrender.com/verify-token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ token: 'JWT_TOKEN_HERE' })
});

const result = await response.json();
if (result.valid) {
  // result.data contains address and score
  console.log("Verified User:", result.data.sub);
  console.log("Trust Score:", result.data.score);
}

API Reference

Endpoint Type Description
/verify-token POST Verifies a JWT token issued by DIDChain.
/score/:address GET Returns the trust score for a specific public address.
/identity/:cid GET Fetches user meta-data from IPFS via CID.
